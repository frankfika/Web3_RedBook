# Web3 黑话表

## 基础概念

| **词条** | **解释** |
| --- | --- |
| Web3.0 | 第三代互联网技术，以去中心化、代币经济和基于区块链的应用为特点，允许用户真正拥有自己的数据和资产，而不仅仅是使用中心化平台所提供的服务。 |
| 区块链 | 一种分布式数据库，在计算机网络上跨多个节点复制和分发数据，确保数据不可篡改和透明度。通过密码学技术保证交易记录的安全性。 |
| 智能合约 | 在区块链上自动执行的代码，没有中央权威机构的参与下实现合约条款的执行。这些程序控制着交易规则、资产转移等操作。 |
| 去中心化 | 将权力从中央权威分散到网络中的所有参与者的过程。在Web3中，指不依赖单一实体控制的系统和应用程序。 |
| 去信任化 | 通过密码学和共识机制，实现不需要相互信任的情况下，参与者也能够安全交互和交易，无需依赖中介或可信第三方。 |
| 可组合性 | Web3应用的关键特性，指不同的应用和协议能够像乐高积木一样组合在一起，创建更复杂和创新的服务，促进生态系统的快速发展和创新。 |

## 身份与安全

| **词条** | **解释** |
| --- | --- |
| 账户抽象 | 一种技术范式，将区块链账户的底层逻辑从区块链层中分离出来，使用智能合约作为可编程钱包，提供增强的安全性和灵活性，简化用户体验。 |
| 自主权身份 | 一种数字身份管理模式，用户掌握自己的身份数据完全控制权，不依赖中央机构。用户可以选择性地向第三方披露信息，同时保留对个人数据的所有权和控制权。 |
| 去中心化标识符 (DID) | 一种全球唯一的标识符，设计用于受用户控制的可验证且去中心化的数字身份。由W3C标准化，使用户可以证明身份所有权而无需中央权威机构。 |
| 可验证凭证 (VC) | 数字化的证明文件，包含加密签名以保证其真实性。类似于物理世界的身份证、驾照或学位证书，但以数字方式存在且可加密验证。 |
| 社交恢复 | 一种账户恢复机制，用户可以指定受信任的"守护者"（朋友、家人或机构），帮助恢复因私钥丢失而无法访问的账户，避免资产永久丢失。 |
| 多签钱包 | 需要多个密钥持有者授权才能执行交易的加密货币钱包。通常用于增强安全性和治理机制，如DAO资金管理或企业级资产管理。 |
| 永生代币 (SBT) | 不可转让的非同质化代币，永久绑定于特定的钱包地址。可用于表示身份、资格证明、成就或会员资格等不适合交易的数字资产。 |
| 零知识证明 | 一种加密方法，允许一方（证明者）向另一方（验证者）证明某个陈述是真实的，而无需透露除陈述真实性之外的任何信息。在Web3中用于隐私保护交易和身份验证。 |
| 零知识证明技术 (ZK) | 允许一方证明自己知道某个事实，而不泄露任何与该事实相关的额外信息的加密方法，被广泛应用于增强区块链的隐私和可扩展性。 |
| 密钥管理 | 在Web3生态系统中，涉及创建、存储、使用和保护密码学密钥的过程。有效的密钥管理对于保障资产安全和身份验证至关重要。 |
| 私钥 | 授权用户访问和控制其加密资产的敏感密钥，必须妥善保管，一旦丢失将无法恢复资产。 |

## 加密技术与隐私

| **词条** | **解释** |
| --- | --- |
| zk-SNARK | 零知识简洁非交互式知识论证(Zero-Knowledge Succinct Non-Interactive Argument of Knowledge)，一种允许在不泄露信息的情况下证明特定计算结果的加密技术。特点是验证时间短、证明大小小，但需要可信设置并且不具备抗量子计算能力。 |
| zk-STARK | 零知识可扩展透明知识论证(Zero-Knowledge Scalable Transparent Argument of Knowledge)，较新的零知识证明技术，不需要可信设置且具有抗量子计算能力，但证明大小相比SNARK更大，验证时间更长。 |
| zkVM | 零知识虚拟机，一种能够生成特定计算正确执行证明的虚拟机技术，允许将通用程序语言编译为能产生零知识证明的格式，简化了零知识应用的开发过程。 |
| 可验证随机函数 (VRF) | 一种加密函数，能产生可被验证为随机生成的输出，常用于区块链中的随机性生成，确保随机性不可预测且可被公开验证。 |
| 多方计算 (MPC) | 一种加密技术，允许多方共同计算函数，而每方只能看到自己的输入和结果。在Web3中用于密钥管理、钱包安全和隐私保护交易，可分散持有私钥的风险。 |
| 链上/链下数据 | 链上数据存储在区块链本身，具有不可篡改性但公开可见；链下数据存储在区块链外部，可保护隐私，但需要额外机制确保真实性。两者结合可实现隐私与验证的平衡。 |
| 数据流动性 | 指用户数据能够在不同应用程序之间自由流动和共享的能力。在Web3中，去中心化存储和身份解决方案使用户能够控制自己的数据，并选择性地授权应用程序访问。 |
| 可验证延迟函数 (VDF) | 一种加密原语，要求在特定时间后才能得出结果，且该结果可被快速验证。在区块链中用于生成不可操纵的随机数和防止某些类型的攻击。 |

## 经济与资产

| **词条** | **解释** |
| --- | --- |
| Depin | DePIN（Decentralized Physical Infrastructure Networks，去中心化物理基础设施网络）是一个创新的网络类型，旨在动员和集合分散的物理资源，如硬盘和算力，以构建庞大的基础设施网络。它分为物理资源网络（PRN）和数字资源网络（DRN），前者专注于提供现实世界的独特商品和服务，后者则致力于数字资源的供给。 |
| RWA | 全称是"Real World Assets"（实体世界资产），指的是在物理世界中存在并具有实际价值的资产通过区块链技术代币化后的表示。不同于资产证券化打通的是传统资产与资本市场之间的桥梁，RWA代币化打通的是现实资产与加密金融之间的桥梁，目的是将现实世界中的资产带到DeFi中来，借助去中心化金融全球可达性的优势给现实资产带来更多的流动性。RWA的底层资产非常广泛，包括但不限于股票、债券、房地产、现金、艺术品、贵金属等。 |
| 代币化 | 将有价值的资产转化为区块链上的数字代币的过程，使其可以在分布式网络上进行交易、分割和编程。这种转化使得传统资产具有了更高的流动性、可分割性和可编程性。 |

## 流动性质押与再质押

| **词条** | **解释** |
| --- | --- |
| 流动性质押 | 一种创新的质押模式，用户可以获得代表其质押资产的衍生代币，既能获取质押奖励，又能保持资产流动性，可用于DeFi活动。流动性质押解决了传统质押中资产被锁定的问题。 |
| 流动性质押衍生品 (LSD) | 代表质押资产的代币，如以太坊上的stETH（Lido）、rETH（Rocket Pool）等。这些代币随着底层质押资产的收益增长而增值，同时可在DeFi中自由使用。 |
| 再质押 | 一项创新技术，允许用户将已质押资产（如ETH）进一步用于质押，以保障其他协议安全并获取额外收益。通过再质押，一项资产可以同时保障多个网络，实现资本效率最大化。 |
| 流动性再质押代币 (LRT) | 代表再质押资产的代币，如eETH（EtherFi）、ezETH（Renzo）等。这些代币结合了流动性质押和再质押的优势，使用户能够在保持资产流动性的同时获得多重收益。 |
| EigenLayer | 一个去中心化的以太坊再质押协议，允许ETH质押者将其质押资产用于保障多个网络和服务的安全。EigenLayer创建了一个市场，使验证者能够选择支持的服务，同时获取额外奖励。 |
| 主动验证服务 (AVS) | 在EigenLayer生态系统中，指利用以太坊质押资产安全性的区块链应用程序，如数据可用性层、去中心化排序器、预言机等。这些服务通过EigenLayer获得ETH质押者的安全支持而无需建立自己的代币和质押系统。 |
| 密码经济安全 | 区块链网络通过经济激励机制提供的安全保障，通常与质押资产的总价值相关。再质押通过允许质押资产同时保障多个网络，实现了加密经济安全的更高效利用。 |
| 削减惩罚 | 在权益证明区块链中对行为不当的验证者实施的惩罚机制。验证者如恶意行为或未履行职责，可能会被削减部分或全部质押资产。对流动性质押和再质押参与者而言，这构成一项风险。 |

## 技术与协议

| **词条** | **解释** |
| --- | --- |
| 主动验证服务 (AVS) | 以太坊再质押生态系统中的区块链应用程序，包括二层网络、数据层、跨链桥等多种服务。AVS通过协议如EigenLayer利用以太坊的验证机制，无需开发自己的共识系统。 |
| 以太坊改进提案 (EIP) | 以太坊网络改进的标准建议文档。EIP-4337是账户抽象的重要提案，引入额外层级实现账户抽象而无需更改以太坊底层协议。 |
| 多方计算 (MPC) | 一种加密技术，允许多方共同计算函数，而每方只能看到自己的输入和结果。在Web3中用于密钥管理、钱包安全和隐私保护交易，可分散持有私钥的风险。 |
| 链上/链下数据 | 链上数据存储在区块链本身，具有不可篡改性但公开可见；链下数据存储在区块链外部，可保护隐私，但需要额外机制确保真实性。两者结合可实现隐私与验证的平衡。 |
| 数据流动性 | 指用户数据能够在不同应用程序之间自由流动和共享的能力。在Web3中，去中心化存储和身份解决方案使用户能够控制自己的数据，并选择性地授权应用程序访问。 |
| 模块化区块链 | 将区块链功能（执行、共识、数据可用性和结算）拆分为独立层次的设计理念，各层可以独立优化和扩展，提高系统整体性能和灵活性。以太坊正逐步向模块化架构转变。 |
| 抗审查性 | 去中心化系统的关键属性，确保没有单一实体能够阻止交易被接受或执行。通过分布式网络和加密技术实现，保障信息和交易自由流通，防止中心化权力干预。 |
| 随机信标 | 生成可验证随机数的机制，用于区块链系统中需要随机性的应用，如抽奖、NFT分配或游戏。设计良好的随机信标应当具备不可预测性、不可操纵性和公开可验证性。 |

## 金融与交易

| **词条** | **解释** |
| --- | --- |
| DeFi | Decentralized Finance（去中心化金融）的缩写，指建立在区块链上的金融应用生态系统，旨在创建一个开放、无许可和无信任的金融服务环境。DeFi应用包括借贷、交易、资产管理和保险等。 |
| DAO | Decentralized Autonomous Organization（去中心化自治组织）的缩写，是一种基于区块链的组织形式，通过智能合约和治理代币进行决策和管理，没有传统的层级结构和中央控制。 |
| NFT | Non-Fungible Token（非同质化代币）的缩写，是区块链上独特的数字资产，每个都具有唯一的标识符和属性，不可互换。常用于代表数字艺术、收藏品、游戏物品和虚拟土地等。 |
| Gas | 在以太坊等区块链网络中执行交易或智能合约所需支付的费用，用于补偿网络节点验证和处理交易的工作。费用根据计算复杂度和网络拥堵情况而变化。 |
| MEV | Maximal Extractable Value（最大可提取价值）的缩写，指矿工或验证者通过重新排序、插入或审查区块中的交易，从区块链用户那里提取的价值。常见于套利、清算和三明治攻击等场景。 |
| TVL | Total Value Locked（锁定总价值）的缩写，指锁定在特定DeFi协议、平台或整个生态系统中的加密货币资产总价值，通常用于衡量DeFi项目的规模和增长。 |
| 聚合器 | 在DeFi中整合多个协议或流动性来源的应用，为用户提供最优交易路径、收益率或服务。例如，DEX聚合器比较多个交易所的价格，提供最佳执行价格和最低滑点。 |
| 收益率农场 | 用户通过提供流动性或锁定代币来获取额外奖励的策略，通常涉及将资金存入智能合约以赚取治理代币或其他激励。是DeFi中常见的代币分发和激励机制。 |
| 闪电贷 | 无需抵押的贷款机制，允许用户在单个区块交易中借入和归还资金，主要用于套利和清算。由于借贷发生在同一交易中，无需传统的信用评估或抵押品。 |

## 基础设施与网络

| **词条** | **解释** |
| --- | --- |
| Layer1 | 区块链的基础层，如比特币和以太坊，负责处理交易并确保网络安全。Layer1解决方案专注于改进基础区块链协议本身的可扩展性、安全性和去中心化程度。 |
| Layer2 | 建立在Layer1区块链之上的扩展解决方案，旨在提高交易吞吐量和降低费用，同时继承底层区块链的安全性。常见Layer2解决方案包括侧链、状态通道、Rollups等。 |
| Rollup | 一类Layer2扩展解决方案，通过将多个交易批处理后提交到主链，大幅提高交易处理能力并降低费用。主要分为乐观Rollup（Optimistic Rollup）和零知识Rollup（ZK-Rollup）两种。 |
| 闪电网络 | 比特币的第二层扩展解决方案，允许参与者在链下创建支付通道进行即时、低成本的交易，只有在通道开启和关闭时才在主链上记录，大大提高了比特币的交易速度和可扩展性。 |
| 互操作性 | 不同区块链网络之间实现无缝通信和价值传输的能力，使各个区块链生态系统能够协同工作，而不是形成孤立的"信息孤岛"。跨链桥接和协议是实现互操作性的常见方式。 |
| 共识机制 | 区块链网络中用于验证交易和创建新区块的算法或协议，确保所有参与节点就网络状态达成一致。常见的共识机制包括工作量证明(PoW)、权益证明(PoS)和委托权益证明(DPoS)等。 |
| IPFS | 全称InterPlanetary File System（星际文件系统），是一种去中心化的文件存储和共享网络，使用内容寻址而非位置寻址，为Web3应用提供持久化和可验证的数据存储解决方案。 |
| 数据可用性 | 区块链系统的关键属性，确保所有网络参与者都能访问和验证区块中的交易数据。在扩展解决方案中尤为重要，是确保系统安全性和防止欺诈的基本要素。 |
| 证明系统 | 在密码学和区块链技术中，一种允许证明者向验证者证明特定计算正确执行的机制，无需重新执行全部计算。包括零知识证明、有效性证明和欺诈证明等多种类型。 |
| 乐观Rollup | 一种Layer2扩展解决方案，它"乐观地"假设所有交易都是有效的，但允许在争议期内提交欺诈证明来挑战无效交易。特点是与EVM高度兼容，但有7-14天的挑战期。 |
| ZK-Rollup | 利用零知识证明技术的Layer2扩展解决方案，通过提交有效性证明来证明链下计算的正确性。相比乐观Rollup，它提供即时终局性，但与EVM的兼容性较为有限。 |
| RANDAO | 以太坊中的随机数生成机制，通过将区块提议者的贡献混合在一起生成随机值，用于协议级随机性。虽然不完全不可预测，但提供了足够安全的随机源。 |

## 社区与文化

| **词条** | **解释** |
| --- | --- |
| WAGMI | "We're All Gonna Make It"的缩写，表达对加密货币和Web3项目未来成功的乐观态度和社区团结精神，常在市场上涨或项目取得重大进展时使用。 |
| NGMI | "Not Gonna Make It"的缩写，表示某人或某个项目可能不会成功或未能把握重要机会，常用于批评不明智的投资决策或错失良机的情况。 |
| DYOR | "Do Your Own Research"的缩写，鼓励投资者在投资前进行独立研究，而不是盲目跟随他人建议。这是加密社区强调个人责任和信息验证的常用术语。 |
| 荷尔德 | 源自"Hold"（持有）的拼写错误，指的是长期持有加密资产的策略，而不受短期市场波动的影响。反映了对项目长期价值的信念和对市场波动的抵抗。 |
| FOMO | "Fear Of Missing Out"（错失恐惧）的缩写，描述投资者担心错过可能的收益而做出冲动决策的心理状态，常导致在价格高点购买和非理性投资行为。 |
| FUD | "Fear, Uncertainty and Doubt"（恐惧、不确定性和怀疑）的缩写，指传播负面信息以影响市场情绪的策略，或描述市场中普遍存在的负面情绪和不信任感。 |
| 空投 | 加密项目向特定用户群体免费分发代币的营销策略，通常用于奖励早期用户、扩大用户基础或提高代币流通。接收者通常需要满足特定条件或完成特定任务。 |
| PUGC | 参与式用户生成内容，指用户通过特定平台创造并分享的内容，而平台提供必要工具和环境。与传统UGC相比，PUGC更强调用户与平台间的互动协作。 |
| 生态系统补助金 | 区块链项目或基金会向开发者、研究者和创业者提供的资金支持，用于鼓励生态系统建设和创新。是Web3中支持公共产品和基础设施的重要机制。 |
| 代币社区 | 围绕特定代币或项目形成的社区，成员通过持有代币获得归属感、治理权和经济激励。代币设计和分配机制对社区发展和凝聚力有重要影响。 |
| 去中心化社会（DeSoc） | 利用区块链、Web3技术和可灵活组合的社会资本创建新型社会结构的愿景，重新定义数字社会中的身份、声誉和关系网络。 |

## 交易与投资

| **词条** | **解释** |
| --- | --- |
| 多头 | 预期资产价格上涨并采取买入策略的投资立场。多头投资者通过在低价买入，高价卖出来获利，反映了对市场或特定资产的乐观情绪。 |
| 空头 | 预期资产价格下跌并采取卖出策略的投资立场。空头投资者借入资产并卖出，希望在价格下跌后以更低价格回购并归还，赚取差价，表达了对市场的悲观预期。 |
| 精英韭菜 | 指那些自认为比普通投资者（韭菜）更有经验和知识，但实际上仍然容易受到市场操纵和亏损的投资者。这个术语带有自嘲色彩，提醒人们保持谦虚和警惕。 |
| 波场 | Token Market Cap排名前10的项目，也引申为大家在高位接了血亏，就比如一顿饭吃了200元你会感觉很贵，但是200元的衣服买起来毫不犹豫，同样是200元对不同事物的价值判断是不同的。 |
| 纸手 | 指在价格稍有波动就急于卖出资产的投资者，缺乏耐心和长期持有的意愿。常被用来批评那些因短期市场波动而放弃长期投资机会的行为。 |
| 钻石手 | 比喻像钻石一样坚硬的手，指在面对极端市场波动时仍然坚定持有资产不卖出的投资者。象征着对投资的坚定信念和抵抗短期市场压力的能力。 |
| 站岗 | 指投资者在资产价格高点买入后，价格大幅下跌，不得不长期持有等待价格回升的状态。形象地描述了被套牢的投资者如同站岗一般被固定在某个位置上的窘境。 |
| 割韭菜 | 形象地描述市场操纵者或项目方利用信息不对称和市场波动，从普通投资者（韭菜）那里获取利益的行为。暗示了这种剥削行为的周期性，就像割了韭菜还会再长一样。 |
| 币圈一天，人间一年 | 形容加密货币市场变化之快，一天内可能经历的波动和事件，在传统市场可能需要一年时间才会发生。反映了加密市场的高波动性和快节奏特性。 |
| 做市商 | 通过提供资产买卖双向报价维持市场流动性的实体，从买卖价差中获利。在加密市场中，既有中心化交易所聘请的专业做市商，也有自动化做市商（AMM）协议。 |
| 预言机 | 连接区块链与外部世界的服务，为智能合约提供链下数据（如价格、天气或事件结果）。解决了区块链无法直接访问外部信息的限制，是DeFi等应用的关键基础设施。 |
| 套利者 | 利用不同市场或协议间的价格差异进行无风险或低风险交易以获利的参与者。套利对维持市场效率和价格一致性至关重要，但也可能导致MEV问题。 |
